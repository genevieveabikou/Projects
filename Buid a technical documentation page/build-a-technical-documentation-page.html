Introduction

The Weather API allows developers to access data from anywhere in the worlds. It provides accurate information such as temperature, humidity, wind speed, and more.

This documentation will help you get started with using the Weather API for your project.

Additionally, the API supports both free and paid tiers, allowing developpers to start with a free plan and scale up as needed, based on usage.

With the proper error handling and authentication, you can retrieve weather data sealessly and securely, making your apps more user-friendly and reliable.

Getting started

To start using the Weather API, you first need to sign up for an account and obtain an API key. The API key will be required in all your requests to authenticate and authorize access to the weather data.

Once you have your API key, you can start making requests to the Weather API by passing it in the request header or as a query parameter.

  • Visit the Weather API Sign Up Page to get your API key.
  • Authenticate using your API key with every request.
Authorization: Bearer YOUR_API_KEY
Authentication

Every request made to the Weather API must include your unique API key for authentication. Requests without a valid API key will result in an authorization error.

If at any time, the API key is compromised, you must regenerate a new key by clicking the Regenerate button located next to the API key.

Alternatively, you can pass the API key as a query parameter:

GET https://api.weatherapi.com/v1/current.json?key=YOUR_API_KEY&q=Location
API Endpoints

The Weather API provides multiple endpoints to fetch data based on location, time, and weather conditions. Below are some API key endpoints:

Current Weather

Fetches real-time weather data for a specific location:

GET https://api.weatherapi.com/v1/current.json

Forecast

Fetches weather forecast data for the upcoming days:

GET https://api.weatherapi.com/v1/forecast.json?key=YOUR_API_KEY&q=Location&days=5
  • q //The location (city or postal code)
  • date //Number of forecast days (up to 10 days)

Search or Autocomplete

Helps to search for a location by name, postal code, or even IP address. This can also be used to provide autocomplete suggestions for user input:

GET https://api.weatherapi.com/v1/search.json?key=YOUR_API_KEY&q=Location

History

Retrives historical weather data for a given location. You can specify the date for which you want the weather data:

GET https://api.weatherapi.com/v1/history.json?key=YOUR_API_KEY&q=Location&dt=2024-09-12

Sports

Gives weather data specifically tailored for sports, such as soccer, golf, or tennis, including the conditions at sports venues.:

GET https://api.weatherapi.com/v1/sports.json?key=YOUR_API_KEY&q=Location
Error Handling

The Weather API provides detailed error messages for every failed request. Common errors include invalid API keys, missing required parameters, and exceeded rate limits.

When an error occurs, the API returns a status code and a detailed error message in the response body, which can help diagnose and resolve the issue quickly.

Here's an example of an error response when an API key is invalid:


                    {
                        "error":{
                            "code":1006, 
                            "message": "API key is invalid or not provided."
                        }
                    }

It is essential to implement proper error handling in your application to gracefully manage failed API requests.

Errors related to exceeded rate limits can be avoided by monitoring your API usage and ensuring your requests stay within the limits of your subscription plan. Considering the implementation of caching to reduce the number of requests is a recommendation.

Here are some few API error codes and their meaning:

  • 1002: API key not provided.
  • 1003: Parameter 'q' not provided.
  • 1005: API request url is invalid.
  • 1006: No location found matching parameter 'q'.
  • 2006: API key provided is invalid.
